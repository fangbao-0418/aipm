AI 原型 & PRD 生成平台 — 需求文档（PRD v1.0）

一、产品定位

1.1 产品目标

构建一个「AI驱动的产品设计平台」，实现：

从自然语言 → PRD → UI原型 → 可编辑 → 生成多端代码

降低产品经理、设计师、开发之间的沟通成本。

1.2 核心价值

需求快速结构化（AI生成PRD）

自动生成页面结构与UI原型

支持可视化编辑（类似轻量墨刀）

支持局部AI调整（而非重生成）

支持多端代码导出（D2C）

1.3 目标用户

产品经理

独立开发者

创业团队

设计师（轻量场景）

功能描述

“做一个电商App首页，包含banner、商品列表、分类入口”

PRD文档

  "prd": {},
  "pages": [],
  "designSchema": {}

2.2 Schema中枢（系统核心）

设计原则

所有内容必须结构化，不允许直接生成代码

Schema结构

{
  "type": "page",
  "children": [
    {
      "type": "banner",
      "props": {}
    }
  ],
  "layout": {
    "x": 0,
    "y": 0
  }
}

能力

统一UI描述

支持跨端渲染

支持AI修改

支持代码生成

2.4 AI局部调整（差异化重点）

功能描述

用户可以针对：

单个组件

某一块区域

某个页面

进行AI修改

示例

用户选中按钮：

“把这个按钮改成渐变色，并加一点阴影”

系统行为

定位组件节点

AI生成修改patch

应用到schema

Patch示例

{
  "targetId": "btn_1",
  "changes": {
    "style": {
      "background": "linear-gradient(...)"
    }
  }
}
