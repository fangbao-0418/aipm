# AI Design Agent 交互规范 v1

## 目标
AI Design Agent 不是单个聊天机器人，而是由负责人、产品经理、UI 设计师、Schema 执行、视觉审查组成的多角色工作流。用户输入需求后，Agent 需要完成需求理解、页面规划、UI 生成、逐页预览、审查修复和产品确认闭环。

## 角色职责

### 负责人 Agent
- 判断任务类型：生成新 UI、修改当前画布、局部组件生成、页面查询、纯问答。
- 输出决策原因：为什么这是新建 UI，为什么是修改当前区域，为什么需要先审查或先读取 schema。
- 负责规划工具链，决定是否继续执行、停止、回滚或要求人工确认。
- 生成 UI 时必须在当前画布右侧追加画板，不创建独立页面；修改任务才允许定位并修改已有节点。

### 产品经理 Agent
- 输出结构化产品信息，而不是只输出“识别到几个功能点”。
- 必须包含：功能点、非功能要求、界面设计要求、交互要求、关键状态、风险和待确认问题。
- 对需求不完整时，先基于合理假设生成第一版，但要把假设写出来。

### UI 设计师 Agent
- 基于产品经理输出生成 UI 方向，包括页面结构、布局规则、组件选择、视觉风格、状态覆盖。
- 移动端基准：750px 设计稿 / 375px 逻辑宽度，单列布局，主操作靠近底部但避开安全区。
- PC 端基准：1920px 宽屏，常用画板宽度 1440px，导航、内容区、表格/表单分区清晰。
- 不允许元素遮挡、剪切、溢出画板、无主操作、无状态反馈。

### Schema 执行 Agent
- 负责把页面设计转换成可编辑 schema。
- 生成 UI 时在当前画布已有画板右侧追加，顶对齐，默认间距 40px。
- 修改 UI 时只能修改当前画布指定区域，必须先读取或分析当前页面结构。

### 审核 Agent
- 每个生成画板都要审查。
- 审查项包括：需求覆盖、页面结构、主操作、空/错/加载状态、元素是否越界、是否重叠、移动端/PC 规范是否符合。
- 发现阻塞问题时，必须给出修复工具建议，并优先自动修复；不能只告诉用户失败。

## 聊天输出规则
- 负责人必须输出“任务判断 + 决策原因”。
- 产品经理必须输出“需求拆解结果”。
- UI 设计师必须输出“设计方案”，不能只输出空标题。
- 工具执行开始只输出一次，避免重复的“准备执行”。
- 生成后必须输出画板预览图，让用户确认是否符合需求。
- 如果能力未接入，必须明确说明是未实现工具，而不是 fallback 成看似成功的结果。

## 工具实现状态
- `requirement.parse`：已实现基础规则解析，需要继续增强 LLM 结构化解析。
- `flow.generate`：已实现页面与流程生成，需要继续支持复杂业务流。
- `asset.resolve`：已实现素材需求识别和策略输出，真实素材库/联网检索仍需接入。
- `schema.generate_ui_from_requirements`：已实现当前画布右侧追加画板。
- `canvas.capture`：已实现 schema 级 SVG 预览，真实浏览器截图和视觉模型审查待接入。
- `ui.review_design`：已实现基础审查，包括搜索区、遮挡、越界、主操作检查，需要继续增强审美与交互检查。
- `web.search`：已注册但未接真实联网搜索。
- `image.to_schema`：已注册但未接视觉模型。

## 生成 UI 流程
1. 负责人判断任务类型，并说明决策原因。
2. 产品经理解析需求，输出功能点、非功能要求、界面要求和交互要求。
3. 产品规划生成页面清单、用户流程和状态。
4. 素材 Agent 输出素材清单和获取策略。
5. UI 设计师输出页面布局、视觉规范和组件策略。
6. Schema 执行 Agent 在当前画布右侧追加画板。
7. 视觉识别 Agent 输出新增画板预览图。
8. 审核 Agent 逐页审查，有问题自动给出修复建议。
9. 负责人要求用户确认是否继续细化、修改或进入下一轮。
